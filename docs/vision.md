# Haera

AI가 널리 쓰이면서 개발자는 여러 프로젝트를 동시에 진행하게 됐다. 그 결과 여러 repo, 여러 사람, 여러 소통 채널에 흩어진 업무 컨텍스트를 관리하는 일이 큰 부담이 되고 있다.

Haera는 흩어진 업무 입력을 실행 가능한 작업으로 정리하고, 코딩 에이전트나 담당자에게 연결하는 운영 시스템이다.

요구사항은 메신저에 있고, 결정은 회의에서 나오고, 참고 링크는 메일에 있고, 구현은 GitHub에서 진행된다. Haera는 메시지, 이메일, 링크, 접속 정보, 회의 내용, 음성 메모, GitHub 이슈 같은 입력을 형식 없이 받아들인다.

Haera는 입력 내용을 파악해 필요한 정보를 정리하고 작업으로 만든다. 이후 코딩 에이전트나 담당자에게 할당하고, 진행 상황과 결과를 추적하며, 중요한 판단이 필요한 순간에만 사람에게 묻는다.

Haera의 목표는 사람이 직접 정리하고, 전달하고, 확인해야 하는 일을 줄이고, 사람의 개입을 최소화하는 것이다.

## Core Loop

1. 입력 수집
2. 자동 구조화
3. 작업 생성
4. 실행 주체 배정
5. 진행 추적
6. 판단 필요 항목 상신
7. 결과 보고

## Why

조직에서 많은 비용은 실제 작업 자체보다, 작업을 시작할 수 있는 상태로 만드는 과정에서 발생한다.

- 여러 채널에 흩어진 요청을 모으기
- 링크와 참고자료 확인하기
- 접속 정보와 관련 문맥 정리하기
- 관련 repo와 담당자 찾기
- 실행 가능한 작업으로 쪼개기
- 사람이나 에이전트에게 전달하기
- 진행 상황과 막힌 이유 추적하기
- 결과를 보고 가능한 형태로 정리하기

Haera는 이 입력, 정리, 전달, 추적 비용을 줄인다.

## Features

- **자유 입력 수집**: 메시지, 이메일, 링크, 음성, 회의, GitHub 이슈를 형식 없이 입력
- **자동 구조화**: task, repo, 사람, 일정, 접속 정보, 후속조치 추출
- **웹링크 해석**: 입력에 포함된 웹링크 내용을 읽고 작업 맥락에 연결
- **작업 배분**: 코딩 에이전트 또는 담당자에게 할당
- **진행 추적**: 진행 중, 막힘, 승인 대기, 완료 상태 관리
- **질문 상신**: 중요한 판단이 필요한 경우 담당자에게 질문
- **결과 보고**: 작업 결과를 요약해 판단 가능한 형태로 보고
- **모바일 연동**: 알림, 승인/반려, 음성 지시, 현재 상황 확인
